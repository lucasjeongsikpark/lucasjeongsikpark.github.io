Constrained Decoding for Robust NL-to-SQL

Report

Overview

Schema-Aware Structured Generation with vLLM, Outlines, and XGrammar

Jan 2026 -- May 2026

Overview

Built a vLLM-based NL2SQL pipeline with schema-aware constrained decoding, evaluating Qwen3-0.6B in both thinking and non-thinking modes across WikiSQL and Spider.

Motivation

Freeform decoding often produces invalid or non-executable SQL queries. This project studies whether decoding-time structural control can improve reliability without retraining the model.

Methods

Compared two constrained decoding methods: XGrammar, a context-free grammar approach, and Outlines, a regular-expression/finite-state method. Constraints were designed on WikiSQL and then transferred to Spider to test generalization.

Evaluation

Measured valid SQL rate, executable rate, execution accuracy, and latency over paired experiments. On WikiSQL, constrained decoding improved SQL validity and execution accuracy. On Spider, the same constraints transferred poorly and could degrade performance, showing the need for distribution-aware decoder design.
